# 프록시 패턴
  실질적인 서비스를 제공하는 객체 대신 프록시 객체를 통해 대리인의 역할을 하도록 하는것. 

## 특징
클라이언트 입장에서는 실제 서비스 객체 사용법과 프록시 사용법이 동일하다는 장점이 있다.
서비스 객체의 결과를 프록시가 그대로 반환해야 하므로 인터페이스를 활용해 동일한 메소드를 구현한다. 

## 종류
1. virtual Proxy : 자원이 많이 필요로 하는 작업이 필요할때 프록시 클래스로 전처리 등 을 진행하고 자원이 많이 필요로 하는 작업을 진행.
2. Protection Proxy : 클라이언트가 서비스객체에 접근하는것에 조건을 걸거나, 제한이 필요로 할때 프록시에서 처리하도록 진행
3. Remote Proxy : 물리적으로 동일한 공간에 존재하지 않는 시스템을 로컬에 있는 프록시를 활용하여 사용.
