# 팩토리 패턴(Factory pattern)

객체가 생성되는 부분과 사용되는 부분을 분리하여 결합도는 느슨하게 하고 특정 객체 구현에 의존성은 낮추는 방법

## 종류
1. 심플 팩토리 패턴
2. 팩토리 메서드 패턴
3. 추상 팩토리 패턴

## 예제
### example.factorymethod
- 팩토리 메서드 패턴으로 구현한 예제
- 신규입사자(Employee) 객체를 사용하는 부분은 Hotel.joinCompany() 메소드에 존재하고, 객체를 생성하는 부분은 추상클래스 Hotel의 createEmp를 확장하여 사용
- Employee 구현 클래스의 종류는 Hotel 클래스가 결정하지 않고  재정의된 createEmp()가 결정
- Hotel 클래스가 Employee 구현클래스에게 의존적이지 않아 확장성과 유연성이 높아짐

### example.abstractfactory
- 추상 팩토리 패턴으로 구현한 예제
- 운영체제별 GUI 종류(Button, TextArea)가 다름
- 운영체제에 따라 서로 연관된 구성품을 사용
- 예제와 같이 객체를 일관된 방식으로 생성하고 그것들을 이용해서 하나의 구성을 만들 때 사용
- 주로 인터페이스를 이용해서 서로 연관되고 의존적인 객체들로 이루어진 제품군을 생성
